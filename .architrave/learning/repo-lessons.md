# Architrave Repo Lessons

Candidate lessons learned while implementing in this repo. Keep this file short.
Each entry needs evidence and validation before promotion. Do not store secrets.
Promote repeated, stable lessons into `architrave.config.json`, `AGENTS.md`, `.github/instructions/`, or docs after review.

## Candidate Lessons

| Lesson | Evidence | Occurrences | Validated | Proposed Target | Status |
|---|---|---:|---|---|---|
| Keep metadata discovery, rights eligibility, and acquisition as separate capabilities; a metadata hit must never enable delivery by itself. | `docs/providers/policy.md`; Phase 0 research and Service Architect report | 1 | yes, design-level | `docs/providers/policy.md` | candidate |
| For SMTP workflows, mail-server acceptance is Submitted; an ambiguous post-send crash must block automatic retry. | ADR-0004; Operations UX and Service Architect reports | 1 | yes, design-level | `docs/architecture.md` | candidate |
| Browser overflow checks alone do not detect fixed-position occlusion; inspect representative pixels and add targeted assertions for each failed visual rule. | Phase 1 navigation overlay and SearchEReader/Confirm PIN judge findings; `scripts/capture-stories.mjs` and story play tests | 1 | yes, implementation-level | `docs/ui/design-spec.md` | candidate |
| On this macOS host, native PostgreSQL can fail even a 56-byte SysV allocation while `ipcs` is empty; run one verified PostgreSQL container, accept either official local image tag, and isolate tests by database instead of starting one cluster per test. | Phase 2 kernel `shmget` probe; `scripts/run-backend-tests.mjs`; 54-test PostgreSQL gate with the convenience tag absent | 1 | yes, host-specific implementation evidence | `.architrave/learning/repo-profile.md` | candidate |
| Test bundled Node entrypoints as child processes: in-process TypeScript tests did not expose externalized CommonJS/ESM interop or CLI stack-trace and pool-cleanup failures. | Phase 2 `ipaddr.js` runtime failure and redacted CLI repair; real SSR `react-dom/server` startup failure; `tests/backend/{cli,web-entrypoint}.test.ts` | 2 | yes, implementation-level | `docs/backend/phase-2-plan.md` | promotion-proposed; approval required |
| A read-then-upsert throttle counter loses concurrent failures before the row exists; serialize the logical throttle key before reading and writing it. | Four concurrent PIN failures persisted as two before `lockThrottle`; `tests/backend/identity.test.ts` | 1 | yes, PostgreSQL integration evidence | `docs/architecture.md` | candidate |
| Plugin code activation belongs to the reviewed deployment inventory, while user-owned account connections belong to auth/mail/destination capability state; mixing them creates unnecessary profile toggles. | User clarification; ADR-0005; Phase 3 enablement-removal diff and tests | 1 | yes, product and implementation evidence | `docs/architecture.md` | candidate |
| A discovery-time plugin digest is insufficient when cache hits bypass execution; verify the current digest before cache reads, queue consumption, and worker execution, and fail closed when files disappear. | Phase 3 cache-integrity review; `pluginDigestIsCurrent`; focused plugin/catalog/delivery tests | 1 | yes, implementation evidence | `docs/security/threat-model.md` | candidate |
| Token reconciliation must compare worktree state before and after generation; comparing the dirty worktree to `HEAD` misclassifies every uncommitted feature as token drift. | Phase 3 closure failure and repaired `gates/reconcile.sh` / `.ps1`; both gates pass on the same dirty tree | 1 | yes, cross-platform gate evidence | Architrave kit `gates/reconcile.*` | candidate |
| When a public reference identifies only the plugin and item, multiple eligible same-plugin capabilities are ambiguous; require exactly one eligible capability or carry the capability ID in the reference, and fail closed until the contract chooses one. | Phase 2 GPT judge finding; `src/modules/plugins/capabilities.ts`; ambiguous-source regression in `tests/backend/plugin-catalog.test.ts` | 1 | yes, implementation-level | `docs/architecture.md` | candidate |
| Binding a cache key to the current plugin digest is necessary but not sufficient: revalidate cached protocol payloads before rendering, and mark any stale fallback explicitly in provider evidence. | Phase 3 Google Books adversarial review; protocol-envelope cache validation and stale/failure regressions in `src/modules/plugins/catalog.ts` and `tests/backend/plugin-catalog.test.ts` | 1 | yes, implementation-level | `docs/security/threat-model.md` | candidate |
| Proving an image allowlist requires aligned context exclusions, explicit build/runtime copies, exact final-filesystem inspection, and inspection of every saved layer; a negative context sentinel needs a positive allowlisted control so an unrelated build failure cannot create a false pass. | Phase 6 `.dockerignore`/Dockerfile boundary; `scripts/check-image-inventory.*`; 21-layer and context-control gate | 1 | yes, implementation-level | release engineering documentation | candidate |
| Deployment-owned profile aliases must remain presentation data: reconcile them transactionally by immutable UUID under a migrator-only guard, and make every serving path fail closed on configuration/database drift. | ADR-0007; `migrations/0007_profile_alias_configuration.sql`; `src/modules/config/profile-parity.ts`; profile/migrator/router/worker tests | 1 | yes, implementation-level | `docs/architecture.md` | candidate |
| A plugin rights claim is not effect authority: bind acquisition to a core-owned reviewed inventory, current digest, exact capability shape, and fresh policy checks at preflight, queue, pre-acquire, and pre-delivery. | ADR-0007; `src/modules/plugins/rights-policy.ts`; forged cache/preflight and worker TOCTOU regressions | 1 | yes, implementation-level | `docs/security/threat-model.md` | candidate |
